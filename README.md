# Source Code Purchase Checklist

A practical checklist for developers, agencies and founders evaluating source-code products before purchasing or deploying them.

Use it for:

* PHP and Laravel scripts
* WordPress plugins
* JavaScript applications
* SaaS starter kits
* Admin dashboards
* Self-hosted software
* Mobile application source code

## Why this checklist exists

Buying source code involves more than checking screenshots and features. Before using a product in production, you should understand its technical requirements, dependencies, license, update policy, security risks and long-term maintenance requirements.

This repository provides a structured review process that can be reused across different types of source-code products.

## Checklist sections

* Product purpose and scope
* Technology compatibility
* Code quality
* Security
* Installation and deployment
* Documentation
* Licensing
* Updates and support
* Seller credibility
* Final purchase decision

## Important notice

This checklist does not replace a professional security audit, legal review or detailed source-code inspection. The buyer remains responsible for validating the product before using it in production.
